## What this project does

AI-powered pipeline that maps free-text job categories to canonical O*NET occupation titles.
Users upload an Excel file; the system returns a corrected Excel + full audit trail.

**Core problem:** humans write "Dev Front", "RRHH", "Desarrollador Backend" — systems
expect "Frontend Developer", "Human Resources Managers", "Backend Developer".

## Architecture: 4 agents in a linear Workflow

<!-- Pattern: Agno Workflow + Steps. NOT Team. Step order is fixed — no dynamic routing needed. -->
<!-- Do not introduce Team pattern without discussion. -->

```
IngestAgent → ValidatorAgent → MapperAgent → AuditWriter
```

| Agent | Responsibility |
|-------|----------------|
| `IngestAgent` | Read Excel with openpyxl, extract unique categories from target column |
| `ValidatorAgent` | Compare against `valid_categories.csv` (O*NET), flag anomalies |
| `MapperAgent` | Propose corrections via rapidfuzz + LLM, output typed Pydantic result |
| `AuditWriter` | Write corrected Excel + audit log sheet + review queue |

## The 7 normalization types

<!-- Domain knowledge — not inferable from code. Shapes every MapperAgent decision. -->

| # | Type | Resolved by |
|---|------|-------------|
| 1 | Typo (`"Fronted Developer"`) | rapidfuzz |
| 2 | Casing/punctuation (`"FULL-STACK DEVELOPER"`) | `pre_processor.py` |
| 3 | Seniority stripping (`"Senior Frontend Dev"`) | `pre_processor.py` |
| 4 | Noise/context (`"Dev - Remoto (Contract)"`) | `pre_processor.py` |
| 5 | Language (`"Desarrollador Backend"`) | LLM |
| 6 | Abbreviation/synonym (`"RRHH"`, `"comercial"`) | LLM |
| 7 | Gender inflection (`"Desarrolladora"`) | LLM |

Types 2–4: `pre_processor.py` runs before rapidfuzz — zero tokens, zero latency. Types 5–7: LLM only.

## Confidence scoring

<!-- Thresholds calibrated against O*NET dataset. Run evaluation/ before any change. -->
<!-- Changing thresholds affects token cost and hallucination rate. Do not change without discussion. -->

```
≥ 0.90      →  auto-correct  (no LLM call, zero token cost)
0.70–0.89   →  LLM evaluates semantic equivalence
< 0.70      →  human-in-the-loop (system NEVER guesses)
```

## Hard invariants

- Agent **never invents a category** — `corrected` must be exact value from `valid_categories.csv`
- All agent output uses `output_schema` + Pydantic v2 — never free-text strings
- `AuditWriter` verifies every correction against the DB before writing Excel
- `rapidfuzz` runs before every LLM call — mandatory pre-filter, not optional
- `needs_review=True` cases go to the review queue — never silently applied
- All agents import model from `infrastructure/llm/provider.py` — never instantiate directly in agent file

## Decisions already made

<!-- Each decision was chosen for a specific reason. Reopen only with a concrete tradeoff. -->

- **Chainlit** over Streamlit/Gradio — visualizes agent steps in real time
- **Agno Workflow** over LangGraph — linear flow, less complexity, audit trail built in
- **Agno traces** over LangSmith — coherent with stack, shows `os.agno.com` in portfolio
- **rapidfuzz as pre-filter** — avoids LLM calls for obvious typos (cost + latency)
- **No Team pattern** — dynamic routing adds no value when step order is deterministic
- **`openai` SDK required** — Agno's `OpenAILike` and `LMStudio` classes depend on it even for local endpoints
- **`AgentOS` wraps FastAPI** — do not create `FastAPI()` manually; `AgentOS.get_app()` returns the app
- **Workflow pattern: `Step` → `Steps` → `Workflow`** — `Steps` groups a named sequence; `Workflow` holds `Steps`

## API lookup

<!-- Training data has a knowledge cutoff. Any library in this stack may have breaking changes -->
<!-- after that cutoff. Always verify before writing code — never assume. -->

**Default rule:** before writing code that calls any library in this stack, look up the current
API signature. Do not rely on training data alone — it may be stale.

**Lookup order:**
1. **Context7 MCP** (`resolve-library-id` → `get-library-docs`) — preferred for all packages
2. **WebSearch** — fallback when Context7 has no entry for the package
3. **Read the installed source** (`uv` installs to `.venv/`) — ground truth when docs are ambiguous

**Packages that require lookup without exception** (APIs change frequently or differ from training data):

| Package | Why |
|---------|-----|
| `agno` | Pre-1.0 → post-1.0 had breaking changes; `Workflow`, `Step`, `Steps`, `Agent`, `output_schema` |
| `chainlit` | Decorators and lifecycle hooks change between minor versions |
| `groq` | SDK evolves alongside model availability |
| `pydantic` | v1 → v2 is a breaking migration; always confirm field syntax and validators |
| `fastapi` | Dependency injection and lifespan API changed in recent versions |
| `openpyxl` | Writer/reader API has had breaking changes across minor versions |
| `rapidfuzz` | Scorer names and import paths changed in v3 |
| `duckdb` | Python API surface changes frequently between minor versions |

If Context7 returns no results and WebSearch is inconclusive, say so explicitly and ask before
writing code — do not guess.
